# AI 로봇팔 컨트롤러

Teachable Machine 이미지 분류 모델을 브라우저에서 불러오고, 웹캠 추론 결과를 라벨별 명령어로 바꿔 **Web Serial API**로 아두이노 로봇팔에 전송하는 수업용 웹앱입니다.

> 서울특별시 중부교육지원청 융합정보영재교육원 「AI 분류 모델로 로봇 팔 조작하기」 수업(중학생, 3차시)을 위해 만들었습니다.

## 화면 구성 (탭 3개)

| 탭 | 내용 |
|---|---|
| 🤖 AI 제어 | 모델 불러오기 → 웹캠 추론 → 라벨-동작 매핑 → 자동 제어 (준비 체크리스트 제공) |
| 🔌 연결·테스트 | 실제/모의 장치 연결, 수동 제어 버튼, 직접 명령 전송, 송수신 로그 |
| 📄 아두이노 코드 | 샘플 스케치 보기·복사·다운로드, 업로드 순서, 수정 가능한 상수 안내 |

## 주요 기능

- Teachable Machine 모델 URL 입력 → 라벨 자동 표시
- 웹캠 실시간 분류 (최대 약 10회/초, 좌우 반전 지원)
- 라벨별 로봇팔 명령 매핑 (모델별로 localStorage에 저장)
- Web Serial 연결 (9600 / 57600 / 115200)
- **모의 장치 모드**: 아두이노 없이 전체 흐름 시험
- 수동 제어 버튼으로 하드웨어 개별 시험
- AI 자동 제어 + 안전장치
  - 확률 임계값 (기본 0.80)
  - 라벨 안정화 시간 (기본 700ms)
  - 명령 쿨다운 (기본 1500ms)
  - 동일 명령 재전송 방지 + 재무장 버튼
  - 장치 ACK/DONE 기반 BUSY 처리
- 항상 접근 가능한 **STOP** 버튼 (자동 제어 끄기 + 대기 명령 제거 + STOP 전송)
- 송수신 로그(최근 100개)와 상태 카드

### 안전 원칙

- 자동 제어는 **기본 꺼짐**입니다.
- 페이지 열기, 모델 로드, 웹캠 시작, 시리얼 연결만으로는 어떤 모터 명령도 보내지 않습니다.
- 서보의 실제 각도와 안전 범위는 웹앱이 아니라 **아두이노 코드**에서 제한합니다.
- 로봇팔이 움직일 때 관절과 집게 사이에 손을 넣지 않도록 지도하세요.

## 권장 브라우저

- **데스크톱 Chrome 또는 Edge** (Web Serial API 지원)
- HTTPS 또는 `localhost` 환경 필수
- Firefox·Safari·모바일 브라우저에서는 시리얼 연결이 비활성화되지만, AI 추론과 모의 장치 모드는 계속 사용할 수 있습니다.

## 로컬 실행 방법

```bash
npm install
npm run dev       # http://localhost:5173
```

기타 명령:

```bash
npm run typecheck # 타입 검사
npm run lint      # 린트
npm test          # 단위 테스트
npm run build     # 프로덕션 빌드 (dist/)
```

## GitHub Pages 배포 방법

1. GitHub 저장소의 **Settings → Pages**로 이동합니다.
2. **Build and deployment → Source**를 **GitHub Actions**로 선택합니다.
3. `main` 브랜치에 push합니다.
4. **Actions** 탭에서 `Deploy to GitHub Pages` workflow가 성공했는지 확인합니다.
5. 배포 URL(`https://<사용자명>.github.io/<저장소명>/`)에 접속합니다.
6. 브라우저에서 카메라와 시리얼 권한을 허용합니다.

> Vite의 `base` 경로는 GitHub Actions의 `GITHUB_REPOSITORY` 환경변수에서 자동으로 유도되므로 저장소 이름을 바꿔도 수정할 필요가 없습니다. 다른 경로에 배포하려면 `VITE_BASE` 환경변수로 지정하세요.

## Teachable Machine 모델 URL 얻는 방법

1. [Teachable Machine](https://teachablemachine.withgoogle.com/)에서 **이미지 프로젝트**를 만들고 학습합니다.
2. **모델 내보내기 → Tensorflow.js → 업로드(공유 가능한 링크)**를 선택하고 **모델 업로드**를 누릅니다.
3. 생성된 공유 링크(`https://teachablemachine.withgoogle.com/models/모델ID/`)를 복사합니다.
4. 웹앱의 모델 URL 입력창에 붙여넣고 **모델 불러오기**를 누릅니다.

## 아두이노 연결 방법

0. (조립 전 1회) [arduino/servo_zero_setup/servo_zero_setup.ino](arduino/servo_zero_setup/servo_zero_setup.ino)를 업로드해 서보 4개를 모두 0도로 초기화한 뒤, 그 상태에서 로봇팔을 조립합니다.
0-1. (조립 후 각도 보정) [arduino/servo_angle_test/servo_angle_test.ino](arduino/servo_angle_test/servo_angle_test.ino)를 업로드하고 시리얼 모니터에서 `S 90`(어깨), `E 45`(팔꿈치), `W 120`(손목), `G 30`(집게), `A 0`(전체)처럼 서보별로 움직여 보며 방향과 각도를 확인합니다. 알아낸 값을 본 스케치의 `HOME_*`, `GRIPPER_*`, `MIN_/MAX_` 상수에 옮겨 적으세요.
1. [arduino/robot_arm_serial_example/robot_arm_serial_example.ino](arduino/robot_arm_serial_example/robot_arm_serial_example.ino)를 Arduino IDE로 열어 보드에 업로드합니다.
2. **Arduino IDE의 시리얼 모니터를 닫습니다.** (포트를 점유하면 웹앱이 연결할 수 없습니다)
3. 웹앱에서 장치 종류를 **실제 아두이노**로 선택하고 통신 속도 9600을 확인합니다.
4. **연결** 버튼 → 포트 선택창에서 아두이노 포트를 선택합니다.
5. 수동 제어 버튼으로 각 동작을 먼저 시험한 뒤 자동 제어를 사용합니다.

### 명령어 목록 (웹앱 → 아두이노, 줄바꿈으로 끝남)

| 명령 | 동작 |
|---|---|
| `HOME` | 기본 자세 |
| `GRAB` | 물체 잡기 (집게 닫기) |
| `RELEASE` | 물체 놓기 (집게 열기) |
| `ARM_UP` | 팔 올리기 (어깨) |
| `ARM_DOWN` | 팔 내리기 (어깨) |
| `WRIST_UP` | 손목 올리기 |
| `WRIST_DOWN` | 손목 내리기 |
| `STOP` | 동작 즉시 중지 |

아두이노 → 웹앱 응답: `READY`, `ACK:<명령>`, `DONE:<명령>`, `ERR:UNKNOWN_COMMAND`, `ERR:BUSY`

### 로봇팔 하드웨어 구성

- SG90 서보 4개: 어깨(D5), 팔꿈치(D6), 손목(D9), 집게(D10)
- 받침대 회전(스테퍼모터)은 사용하지 않습니다.
- **조립 기준**: 조립 전에 모든 서보를 0도로 세팅한 뒤, 어깨·팔꿈치·손목 3개 축이 평행이 되도록(0도에서 팔이 곧게 펴진 상태) 조립합니다. 집게 서보만 수직으로 장착합니다.

### ⚠ 외부 전원 및 공통 GND

- 서보 4개는 **반드시 외부 5V 전원**으로 구동하세요. 아두이노 5V 핀만 사용하면 보드가 리셋되거나 고장 날 수 있습니다.
- **외부 전원의 GND와 아두이노 GND를 반드시 공통 연결**하세요.
- 예제 스케치의 핀 번호, HOME 각도, 관절별 최소·최대 각도, 집게 열림/닫힘 각도는 **예시값**입니다. 스케치 상단 상수만 수정하면 되니 실제 로봇팔에 맞게 조정한 뒤 사용하세요.

## 시리얼 모니터 충돌 해결

"포트를 열 수 없습니다" 오류가 나오면:

1. Arduino IDE의 시리얼 모니터/시리얼 플로터를 닫습니다.
2. 같은 포트를 사용하는 다른 프로그램(다른 브라우저 탭 포함)을 닫습니다.
3. USB를 뽑았다가 다시 꽂고 **연결**을 다시 누릅니다.

## 모의 장치 모드 사용법

1. 장치 종류에서 **모의 장치**를 선택하고 **연결**을 누릅니다.
2. 실제 포트 선택 없이 연결되며, 화면에 모의 장치 모드임이 표시됩니다.
3. 명령을 보내면 300~800ms 뒤 `ACK:`, `DONE:` 응답이 자동 생성됩니다.
4. 수동 제어와 자동 제어의 전체 흐름을 아두이노 없이 시험할 수 있습니다.

## 자주 발생하는 오류

| 상황 | 해결 방법 |
|---|---|
| 모델을 불러오지 못함 | Teachable Machine의 **공유 링크**(업로드된 모델 URL)인지 확인 |
| 웹캠 권한 거부 | 주소창 오른쪽 카메라 아이콘에서 권한 허용 후 다시 시도 |
| 아두이노 연결 버튼 비활성화 | Chrome/Edge 사용 여부, HTTPS 또는 localhost 여부 확인 |
| 포트를 열 수 없음 | Arduino IDE 시리얼 모니터 닫기 |
| USB 분리됨 | 자동 제어가 자동으로 꺼집니다. 다시 연결 후 사용 |
| 자동 제어가 켜지지 않음 | 모델·웹캠·장치 연결·라벨 매핑(NONE 아님) 네 가지 조건 확인 |
| 로봇팔이 반복 동작함 | 임계값을 높이거나 안정화 시간·쿨다운을 늘리기 |

## 개인정보 처리 원칙

- 회원가입·로그인이 없고 학생 이름을 입력받지 않습니다.
- 웹캠 영상과 AI 추론은 **모두 브라우저 안에서** 처리되며 서버로 전송·저장되지 않습니다.
- 저장되는 것은 설정값(모델 URL, 매핑, 임계값 등)뿐이며 브라우저 localStorage에만 남습니다.
- 분석 도구와 광고 스크립트를 사용하지 않습니다.

## 수업 운영 순서 (3차시)

| 차시 | 활동 |
|---|---|
| 1차시 | Teachable Machine에서 이미지 분류 모델 학습, 모델 URL 준비 |
| 2차시 | 모델 불러오기 → 웹캠 추론 확인 → 라벨-명령 매핑 |
| 3차시 | 시리얼 연결 → 수동 시험 → 자동 제어 → 임계값·안정화·쿨다운 조정 미션 |

교사 점검 순서: ① 모의 장치로 웹앱 자체 확인 → ② 실제 장치 연결 후 수동 제어 시험 → ③ 로봇팔 주변 안전 확인 후 자동 제어 시작. 문제가 생기면 상태 배지와 로그를 먼저 확인하고, 위험하면 **STOP**을 누르세요.

## 기술 스택

Vite · TypeScript · Vanilla DOM · TensorFlow.js · Web Serial API · Vitest · ESLint · GitHub Actions

> 참고: `@teachablemachine/image` npm 패키지는 TensorFlow.js 1.x에 고정되어 있어 사용하지 않고, `@tensorflow/tfjs` 4.x로 모델 로딩(224×224 센터 크롭, [-1, 1] 정규화)을 직접 구현했습니다.
